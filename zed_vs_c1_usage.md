## 2. 実行コマンド（引数）の違い

C1 デュアルカメラ版では、V4L2 デバイスパス（`/dev/video*`）を明示的に指定できる追加オプションが存在します。

### ■ 直接送信モード (`--send`) の例

**ZED カメラ:**
```bash
./OrinVideoSender --send --server 192.168.1.10 --port 12345
```
*(※自動でZEDカメラが認識されます)*

**C1 デュアルカメラ:**
```bash
./OrinVideoSender_C1 \
  --stereo \
  --device-left /dev/video0 \
  --device-right /dev/video1 \
  --send --server 192.168.1.10 --port 12345
```
*(※`--stereo`フラグと、左右の `/dev/video*` デバイスの指定が必要です)*

### ■ 待受モード (`--listen`) の例

**ZED カメラ:**
```bash
./OrinVideoSender --listen 0.0.0.0:9000
```

**C1 デュアルカメラ:**
```bash
./OrinVideoSender_C1 --listen 0.0.0.0:9000
```
*(※待受モード自体のコマンドは共通ですが、XR等からの接続時に違いがあります。以下参照)*

## 3. クライアント（XR・PC）からの接続仕様の違い

`--listen` モード運用時、受信側（TCP クライアント）から送信する設定プロトコル (`OPEN_CAMERA`) の内容が変わります。

| プロトコル内の設定 | ZED カメラ | C1 デュアルカメラ |
|-------------------|-----------|-------------------|
| **`camera` 文字列** | `"ZED"` | **`"C1"`** |
| **[width](file:///c:/Users/taise/OneDrive/%E3%83%89%E3%82%AD%E3%83%A5%E3%83%A1%E3%83%B3%E3%83%88/%E3%83%AD%E3%83%9C%E3%83%83%E3%83%88%E3%83%90%E3%82%A4%E3%83%88/XRoboToolkit-Orin-Video-Sender/c1_camera.hpp#115-116) (幅)** | ZED の合成解像度 <br> (例: `2560` = 1280x2) | **単眼の解像度** <br> (例: `1280` 指定 → 自動で 2560x720 送信) |

> [!WARNING]
> C1 プログラムに対して `"ZED"` の接続要求を送ると、対象外エラーとして拒否（無視）されます。受信プログラム側の改修も忘れずに行ってください。

---

## 4. 映像・ハードウェア仕様の違い

ハードウェアが異なることによる、機能面での明確な違いです。

| 特徴 | ZED カメラ | C1 デュアルカメラ |
|------|-----------|------------------|
| **映像フォーマット** | 最初から SBS（横並び）で 1 枚の画像として出力される | 2つ別々に取得した画像を、プログラム内で合成（`cv::hconcat`）して SBS 化 |
| **フレーム同期** | **ハードウェア同期（完全）** <br> 左右レンズが同一タイミングで露光 | **ソフトウェア同期（非完全）** <br> 左→右の順に画像を取得するため、ミリ秒レベルのズレ（ローリング等）が生じる可能性あり |
| **キャリブレーション** | **工場出荷時に補正済み** <br> 歪みや平行度が保証されている | **未補正（Raw）** <br> 樽型歪みや、カメラを並べた際の物理的なズレがそのまま映像に出る |
| **自動露出などの追従** | 左右レンズで連動して行われる | カメラごとの独立した自動露出となるため、左右の明るさや色味が合わない場合がある |

### C1 デュアルカメラ特有の注意点（まとめ）
1. C1 カメラを 2 台並べてステレオにする場合、ソフトウェアでフレームを並べているだけなので、**激しく動くシーンでは左右の像にズレ（時間差）** が発生します。
2. また、レンズの歪み補正が行われないため、XR デバイス等で見た際に**視界の端が歪んで見える（VR酔いの原因）** 可能性があります。
