# C1 デュアルカメラ（ステレオ）使用手順書

本プログラム (`OrinVideoSender_C1`) を用いて、C1 カメラ 2 台からステレオ映像（Side-by-Side）を配信する際の手順と注意点をまとめました。

---

## 1. カメラデバイスの確認

Jetson Orin（または Linux PC）に 2 台の C1 カメラを USB 接続し、デバイスとして正しく認識されているか確認します。

ターミナルで以下のコマンドを実行します。
```bash
ls -l /dev/video*
```
通常、`/dev/video0` や `/dev/video1` といったデバイスファイルが表示されます。（環境によっては `/dev/video0` と `/dev/video2` のように飛ぶこともあります）

左目用と右目用のカメラがどのデバイスパスに対応しているかを事前に確認しておいてください。

---

## 2. プログラムのビルド

C1 カメラ対応版のビルドターゲット `c1` を指定してコンパイルします。

```bash
cd /path/to/XRoboToolkit-Orin-Video-Sender
make c1
```
成功すると、実行ファイル `OrinVideoSender_C1` が生成されます。

---

## 3. プログラムの起動方法

当プログラムには、**「直接送信モード」**と**「コマンド待受モード」**の 2 種類の運用方法があります。

### パターン A: 直接送信モード (`--send`)
送信先の IP アドレスとポートが固定で分かっている場合に即座に配信を開始するモードです。

```bash
./OrinVideoSender_C1 \
  --stereo \
  --device-left /dev/video0 \
  --device-right /dev/video1 \
  --send \
  --server 192.168.123.3 \
  --port 12345
```
*(※ `192.168.1.10` の部分は、実際の受信側 PC / XR ヘッドセットの IP アドレスに変更してください。)*

### パターン B: コマンド待受モード (`--listen`)
受信側（クライアント）からの接続と `OPEN_CAMERA` コマンドを待ってから配信を開始するモードです。（XR アプリ等と連携する場合に標準的です）

```bash
./OrinVideoSender_C1 --listen 0.0.0.0:9000
```
*(※ このモードのときは、後述するクライアント側からのコマンド送信時にステレオの設定が適用されます。)*

---

## 4. クライアントからの接続仕様（重要）

`--listen` モードで運用する場合、クライアント（XR ヘッドセット等）から送信する `OPEN_CAMERA` パケットに以下の設定を含める必要があります。

1. **カメラ種別の指定**
   - 従来 `"ZED"` と指定していた部分を、**`"C1"`** に変更して送信してください。
2. **解像度の指定**
   - 配信する解像度は、単眼の解像度を指定してください。（例: 幅 `1280`、高さ `720`）
   - プログラム内部で `--stereo` 動作が指定されていると、自動的に「横に2つつなげた解像度（2560x720）」で配信されます。

---

## 5. 追加のオプション一覧 (C1 専用)

実行時に細かな挙動を制御するためのオプションです。

| オプション | 説明 |
|---|---|
| `--stereo` | **(必須)** 2台のカメラを使用したステレオモードを有効にします。 |
| `--device-left PATH` | 左カメラのデバイスパスを指定します。（デフォルト: `/dev/video0`） |
| `--device-right PATH` | 右カメラのデバイスパスを指定します。（デフォルト: `/dev/video1`） |
| `--width W` | 単眼あたりのキャプチャ幅を指定します。（デフォルト: `1280`） |
| `--height H` | 単眼あたりのキャプチャ高さを指定します。（デフォルト: `720`） |
| `--fps FPS` | キャプチャするフレームレートを指定します。（デフォルト: `30`） |
| `--bitrate BPS` | 動画のエンコードビットレートを指定します。（デフォルト: `4000000` / 4Mbps） |
| `--hevc` | H.264 の代わりに H.265 (HEVC) エンコードを使用します。 |

---

## 6. 既知の制限事項（ZED との違い）

- **完全なハードウェア同期ではない**: 左・右の順番でフレームをキャプチャするため、ごくわずかな時間差（ミリ秒単位）が生じます。高速に移動する物体を見た時に、左右でわずかなズレを感じる場合があります。
- **キャリブレーションの欠如**: C1 カメラは未キャリブレーションの状態です。レンズの歪み（魚眼など）や、物理的なカメラの並び方のズレがそのまま映像の視差のズレとして表れます。ヘッドセット上で見え方に違和感がある場合は、カメラの物理的な固定を精密に行うか、受信側で歪み補正処理を挟む必要があります。
