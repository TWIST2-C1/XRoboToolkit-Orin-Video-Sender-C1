# C1カメラ ステレオ対応 導入・使用ガイド

本資料は、従来のZEDカメラを用いた映像配信プログラムを、汎用的なC1カメラ2台で同等のステレオ配信（Side-By-Side）を行えるように変更した内容とその使用方法をまとめたものです。

## 1. プログラムの変更点

*   **`c1_stereo_camera.hpp / cpp` の新規作成**:
    *   OpenCVの `VideoCapture` を用いて、2台のC1カメラ (V4L2デバイス) から同時にフレームを取得し、左右に結合 (Side-By-Side) して擬似的なステレオ画像を生成するラッパークラスを作成しました。
*   **`main_c1_stereo_tcp.cpp` の新規作成**:
    *   `main_zed_tcp.cpp` をベースとし、ZED SDKのエラーハンドリングやカメラ起動処理を `C1StereoCamera` クラスを用いた処理に置き換えました。
    *   実行時引数に `--cam1` および `--cam2` オプションを追加し、柔軟にデバイスID (例: `/dev/video0`) を指定できるようにしました。
*   **`Makefile` の更新**:
    *   ビルドターゲットのソースコードを `main_zed_tcp.cpp` から `c1_stereo_camera.cpp` および `main_c1_stereo_tcp.cpp` に変更しました。

## 2. プログラムの使用方法

### コンパイル方法
端末を開き、以下のコマンドを実行してビルドします。
```bash
make clean
make
```

### 実行方法
OrinVideoSenderを実行する際、カメラデバイスのインデックスを指定して起動します。
基本コマンド（デフォルトでは cam1=0, cam2=2 が選択されます）:
```bash
./OrinVideoSender --listen 0.0.0.0:8000
```

カメラのデバイスID (`/dev/videoX`の数値部分) を明示的に指定する場合:
```bash
./OrinVideoSender --listen 0.0.0.0:8000 --cam1 0 --cam2 2
```
*   `--cam1`: 左目用カメラのデバイスインデックス
*   `--cam2`: 右目用カメラのデバイスインデックス

### 受信側 (クライアント)からの要求
送信側に対してネットワーク越しにパラメータを送る際、**カメラタイプは `"C1"` を指定**してください（従来は `"ZED"`）。
解像度を指定すると、カメラ1台あたり「指定された幅の半分の解像度」で取得し、結合して指定解像度のSide-by-Side画像を生成します。

## 3. ZEDカメラとC1カメラ(2台構成)の違い

ステレオ配信を行うにあたり、これら2つのカメラシステムには以下の決定的な違いがあります。

| 比較項目 | ZEDカメラ | C1カメラ (2台構成) |
| :--- | :--- | :--- |
| **同期方式** | **ハードウェア同期**<br>左右の画像センサーが物理的・電気的に完全に同期しており、取得時間のズレ（タイムラグ）がありません。高速で動く物体の立体視に優れます。 | **ソフトウェア同期**<br>2つの独立したカメラへOpenCVから順次「撮影」のコマンドを送るため、数ミリ秒〜数十ミリ秒のタイムラグが生じます。 |
| **SDK/API** | ZED専用の強力なSDKが用意されており、深度計算（Depth）や自己位置推定などが標準で扱えます。 | 汎用のV4L2 (Video4Linux2) とOpenCVを用いて映像を取得します。深度計算等は別途ソフトウェアで実装する必要があります。 |
| **画角(FOV)** | モデルによりますが、一般的に広角でステレオ視に最適化された広角レンズが搭載されています。 | 使用するC1カメラのレンズ仕様に依存します。左右で個体差が生じる場合、画角や色合いに微小なズレが出る可能性があります。 |
| **出力形式** | カメラ側で1つのSide-By-Side画像としてきれいにプレアライメントされたフレームを出力します。 | 2つの独立した画像をPC等 (Jetson Orin側) で結合(`cv::hconcat`)するため、CPU/メモリ負荷が少し高くなります。 |

### 運用上の注意点
**ロボットの進行方向や素早い動きに対して、左右の映像に細かな時間の「ズレ」が生じ、VR等で立体視した際に違和感（酔い）を感じる可能性**があります（ソフトウェア同期の限界によるもの）。これが運用上問題になる場合は、ハードウェア同期が可能なステレオカメラ（ZEDなど）の利用を再度検討する必要があります。
